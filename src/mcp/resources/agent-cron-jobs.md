# Agent cron jobs

Scheduled agent tasks stored under `agent_llm_config/cron/jobs.json`. Each job gets a fixed **conversationId**; every run appends to that thread.

## MCP tools

### Read (GET)

- `list_cron_jobs` ‚Äî summaries only (no message body)
- `get_cron_job` ‚Äî full job by **id** or **name** (includes message)
- `list_cron_job_runs` ‚Äî recent run history for **jobId** (optional **limit**, default 50)

### Write (management-signed POST, preferred Ed25519 signer)

- `add_cron_job` ‚Äî create job (**name**, **message**, **schedule**; optional **enabled**, **deleteAfterRun**)
- `update_cron_job` ‚Äî update schedule/message/metadata only (not **enabled**)
- `activate_cron_job` / `deactivate_cron_job` ‚Äî enable or disable without deleting
- `remove_cron_job` ‚Äî delete job; optional **deleteConversation** (default false)
- `run_cron_job` ‚Äî manual async trigger (works when deactivated)

## Schedule kinds

| kind | fields | behavior |
|------|--------|----------|
| `cron` | `expr` (5-field), optional `tz` (default UTC) | clock-anchored cron |
| `every` | `everyMs` | fixed interval from activation |
| `at` | `at` (RFC3339) | one-shot; defaults **deleteAfterRun: true** |

**`add_cron_job` schedule shorthands** (coerced automatically):

- `"every 5 minutes"`, `"5m"`, `"5 minutes"` ‚Üí `{ "kind": "every", "everyMs": 300000 }`
- `"*/5 * * * *"` (5-field cron string) ‚Üí `{ "kind": "cron", "expr": "...", "tz": "UTC" }`
- `300000` (number) ‚Üí `{ "kind": "every", "everyMs": 300000 }`

Prefer structured objects when possible; use shorthands only when simpler for the caller.

## Suggested workflow

1. **`list_cron_jobs`** ‚Äî inspect schedules, enabled state, last/next run.
2. **`get_cron_job`** ‚Äî read the instruction message before editing.
3. **`add_cron_job`** or **`update_cron_job`** ‚Äî set or change the agent prompt and schedule.
4. **`activate_cron_job`** / **`deactivate_cron_job`** ‚Äî pause or resume scheduling.
5. **`run_cron_job`** ‚Äî test immediately; then **`list_cron_job_runs`** for outcome.
6. **`remove_cron_job`** when retiring a task.

## Notes

- Scheduler can be disabled node-wide via `EnableAgentCron: false` or `MPC_AUTH_ENABLE_AGENT_CRON=0`; CRUD and manual runs still work.
- Cron runs use the full agent turn but fail if MCP elicitation would block on human input.
- Job **name**: lowercase `a-z`, digits, hyphen, underscore; max 64 chars.

## Trade analysis cron (optional)

For scheduled multi-analysis + optional multisign, copy the **`trade_analysis_cron.example.md`** template from **mpc-config** (`agent_llm_config.defaults/cron/`). The cron **`message`** combines:

- Prose analysis steps and **selection guidance** (agent picks `tradeIdeaId`)
- Optional fenced **`tradeConsensus`** YAML (`submitTradeFromConsensus: true` enables **`submit_trade_from_consensus`** ‚Äî cron-only)
- Optional fenced **`tradeBuild`** YAML ‚Äî defaults for **`build_trade_from_*`** / **`submit_trade_from_consensus`** (protocol, chain, sizing, gas, market). Example block in the cron message:

````
tradeBuild:
```yaml
protocolId: hyperliquid
chainId: 999
tradeIdeaNumber: 1
szHuman: "0.5"
marketKind: perp
useCustomGas: false
purposeText: Generated by cron.
```
````

Fields: `protocolId`, `chainId`, `tradeIdeaNumber`, `szHuman`, `sizeUsdHuman`, `collateralToken`, `collateralAmountHuman`, `marketKind` (`perp`|`spot`), `tif`, `orderKind` (`market`|`limit`, Uniswap mainnet), `useCustomGas`, `purposeText` (additional suffix appended to auto-generated trade summary; defaults to **`Generated by cron.`** when omitted). Total **`purposeText`** on the multisign request must stay within **256 characters** (auto summary + suffix).

## Uniswap TP/SL monitor cron

For spot Uniswap exits when price crosses analysis TP/SL levels (best-effort agent polling, not on-chain orders):

- MCP: **`register_uniswap_tpsl_monitor_cron`** ‚Äî wraps `add_cron_job` with a standard monitor prompt
- Helper: **`evaluate_uniswap_tpsl_monitor`** ‚Äî used inside the cron message
- Template: **`uniswap-tpsl-monitor-cron.example.md`**

Robinhood Chain (4663) monitors should fetch OHLCV via Bitquery (`BITQUERY_API_KEY`). Disable with **`deactivate_cron_job`** when filled or cancelled.

Interactive chat and orchestrator Continue use **`build_trade_from_*`** instead ‚Äî never `submit_trade_from_consensus` outside `[Cron]` jobs.
